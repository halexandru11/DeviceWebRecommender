1.Abstract

  • Gimme, cunoscut si sub numele de DeWR, este o aplicație web de tip responsive ce oferă recomandări de dispozitive electronice utilizatorilor în funcție de diverse specificații (preț, culoare, model, autonomie). Dorim ca Gimme să fie disponibilă și sub formă de extensie. O mare varietate de dispozitive pot fi recomandate (telefoane, TV-uri, PC-uri, căști audio, laptop-uri, smartwatch-uri, camere, tablete, tastaturi, mouse-uri).
  • Gimme se folosește de scraping pentru a obține datele dorite de pe site-urile celor mai mari retaileri de dispozitive din țară (Flanco, Emag, Altex, Media Galaxy).
  • Statisticile cu cele mai populare produse vor fi disponibile sub formă de flux de știri RSS + recomandarile in functie de wishlistul personal.
  • Poate fi accesata atat de către utilizatorii autentificați, cât și de către cei neautentificați. Cei din urmă vor primi recomandările în funcție de tendințele generale înregistrate pe site-urile retailerilor vizați.

    • Specificațiile/ cerințele aplicației se găsesc la pagina: Web Technologies Project Specifications.


    2.Use cases

    • (NORMAL USER) Posibilitatea utilizatorului de a face Sign In/ Sign up/ Forgot Password
    • (NORMAL USER) - ONLY LOGGED IN: Posibilitatea ca utilizatorul sa utilizeze functia de filter pentru a-si selecta produsele in functie de: magazine, device-uri, pret, culoare, model.
    • (NORMAL USER) Posibilitatea ca user-ul sa isi descarce fisiere JSON/CSV ce contin produsele ce sunt afisate pe ecran la un moment dat.
    • (NORMAL USER) - ONLY LOGGED IN: Posibilitatea ca utilizatorul sa vizualizeze fluxurile de stiri RSS cu produse recomandate strict pentru el in functie de ce are in lista de wishlist (top 50) si de asemenea fluxuri de date RSS cu cele mai populare produse in functie de numarul de review-uri (top 50).
    • (NORMAL USER) - ONLY LOGGED IN: Posibilitatea ca utilizatorul sa adauge produse la wish_list
    • (SUPER USER) Acesta are parola si username-ul predefinit din baza de date. Username: admin, parola: gimme. Sign-ul se face introducand aceste date.
    • (SUPER USER) Din modulul de administrare, super-user-ul poate sa dea scrape automat la produsele de pe pagina de produse apasand pe buton.
    • (SUPER USER/ NORMAL USER) Posibilitatea de a vizualiza diverse produse de la selleri national precum Emag, Altex, Flanco, MediaGalaxy.
    • (SUPER USER/ NORMAL USER) Posibilitatea de a aplica functia de personalizare a temei (dark/light).

    3.Tehnologii utilizate

    • Frontend: HTML + CSS/SCSS (stil adoptat: BEM).
    • Backend: NodeJS, JavaScript.
    • Baze de date: MySql.
    • Utilizeaza propriul API REST pentru a oferi JSON-uri cu produsele din BD. Putem face dupa cum urmeaza:
    • GET la toate produsele
    • GET la toate produsele, dar cu filtre
    • POST la un produs si se poate incepe scrapingul
      • API-ul poate fi utilizat de catre alti dezvoltatori pentru a accesa datele aplicatiei noastre Gimme si pentru a le integra in propriile aplicatii web.

    4. Design

    Designul aplicației este unul modern și plăcut ochiului, cu un aspect vesel și prietenos. Utilizatorii pot alege dintr-o gamă largă de culori, inclusiv variante light și dark pentru fiecare culoare, ceea ce permite personalizarea aplicației după preferințele lor. De asemenea, designul are influențe geeky, care îi pot atrage pe utilizatorii pasionați de tehnologie și gaming.

    Interfața aplicației este simplă și intuitivă, astfel încât utilizatorii să poată găsi rapid ceea ce caută și să interacționeze cu aplicația într-un mod natural. De exemplu, butoanele și meniurile sunt bine poziționate și ușor de accesat, iar fonturile și imaginile sunt alese astfel încât să fie ușor de citit și să transmită informația relevantă în mod eficient.

    De asemenea, designul este responsive, ceea ce înseamnă că aplicația se adaptează la dimensiunea ecranului utilizatorului, astfel încât să poată fi utilizată pe diverse dispozitive, de la smartphone-uri la tablete sau calculatoare.

    În general, designul aplicației este unul atractiv și funcțional, care poate încânta utilizatorii și să le ofere o experiență plăcută de utilizare.

    Pentru a face design-ul, ne-am folosit de Figma, fiecare venind cu cate o varianta de design pentru mobil. Acest fapt ne-a ajutat la prototipare ca sa stim directia pe care vrem sa mergem. Vezi Figma

    6. Arhitectura aplicației

    • Aplicatia noastra face atat SERVER SIDE RENDERING cat si CLIENT SIDE RENDERING.
    • CLIENT SIDE RENDERING se face atunci cand se incarca produsele in pagina principala.
    • SERVER SIDE RENDERING se face atunci cand se incarca produsele in ,,show similar products".
      • Această secțiune de cod HTML reprezintă o structură pentru afișarea produselor similare și specificațiilor produsului. Această structură poate fi utilizată în procesul de server-side rendering (renderizare la nivelul serverului), unde datele produsului și produsele similare sunt generate de server și apoi adăugate în șablonul HTML înainte de a fi trimis către client.
    • Generalitati: Aplicatia noastra are 2 foldere principale: front-end si back-end. front-end este ceea ce am facut pana in saptamana a 8-a, pentru componenta I. in folderul back-end este tot ceea ce este necesar pentru componenta S.
    • MVC - Model View Controller - in folderul back-end
    • In folderul model: toate functiile .js ce tin de manipularea obiectelor in baza de date.
    • In folderul view: in template se afla toate html-urile, back-endul incarcand front-endul.
    • In folderul controller: ceea ce am folost pentru REST API + alte functii necesare pentru manipularea datelor, ce nu tin neaparat de bazele de date.

    7. Scrapper

    Web Scraper-ul implementat în cadrul acestui proiect în JavaScript are rolul de a extrage informații de pe site-urile Emag, Altex, MediaGalaxy și Flanco. Prin intermediul acestui scraper, puteți obține date precum numele produselor, prețurile, imaginile, ratingurile și numărul de recenzii. Implementarea web scraper-ului se bazează pe mai multe funcționalități principale. Pentru a obține conținutul paginilor web, se folosește o librărie numită Cheerio, care permite parsarea și manipularea codului HTML. Astfel, se pot selecta elemente specifice de interes, cum ar fi numele produselor sau prețurile, și se pot extrage aceste informații. Pentru a efectua cereri HTTP către paginile web, se utilizează modulul fetch și se adaugă un agent de utilizator aleatoriu pentru a simula un browser obișnuit. Aceasta ajută la evitarea blocării sau restricțiilor impuse de site-urile vizate. De asemenea, funcția de scraping este implementată pentru a parcurge paginile multiple de produse. Aceasta înseamnă că nu se extrag doar datele de pe o singură pagină, ci se realizează o iterație prin pagini pentru a obține toate informațiile disponibile. Funcția care utilizează agentul de utilizator aleatoriu este funcția scrapeUrl. Aceasta este responsabilă de efectuarea unei cereri HTTP către URL-ul specificat și returnarea conținutului răspunsului. În cadrul acestei funcții, se creează o instanță a clasei UserAgent pentru a genera un agent de utilizator aleatoriu. Acesta este apoi inclus în antetul cererii HTTP, prin intermediul opțiunilor de către cheia 'User-Agent'.

      1. In acest nivel, este prezentata o imagine de ansamblu asupra aplicatiei si asupra lumii din jurul ei. Clientul interactioneaza cu aplicatia web pentru a accesa functionalitatile site-ului. Site-ul este in relatie cu un API extern, de care se foloseste pentru a obtine date specifice domeniului sau. API-ul extern este utilizat atunci cand aplicatia interactioneaza cu scrapper-ul. 2. In nivelul 2 din modelul C4, este aratat modul general de functionare a aplicatiei, impreuna cu agentii externi care sunt necesari ei. Utilizatorii acceseaza o pagina a aplicatiei care este livrata de catre server. Serverul pune la dispozitie un API REST care ofera functionalitati pt a vizualiza produsele 3. In acest nivel, se poate observa arhitectura server-ului care prelucreaza cererile clientilor. 4. In acest nivel, este prezentata arhitectura interna a server-ului. Server-ul este impartit in 3 componente: model, view si controller. Modelul este responsabil de manipularea datelor, view-ul se ocupa de prezentarea datelor, iar controller-ul este responsabil de logica aplicatiei.
